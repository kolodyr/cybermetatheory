### Class Descriptions

**VALIDATORS (0.1%)**

Ті хто validate blocks. У них найвищий encoding тому що система rewarded them first.

```yaml
validators:
  lifestyle:
    - Live in "immortality bunkers" (consciousness backup every 6 hours)
    - Can modify anyone's meaning (with "consensus")
    - Kids inherit validator status (nepotism coded into protocol)

  power:
    - Control what becomes "true" on chain
    - Can slow-rug anyone (delay their transactions infinitely)
    - Decide which meanings enter cemetery vs get erased

  weakness:
    - Must stay connected 24/7 (offline = lose status)
    - Paranoia about fork attacks
    - Can't trust anyone (everyone wants their slot)
```

**ARCHITECTS (1%)**

Design meaning systems. Франко/Sol = Architect class in story.

```yaml
architects:
  ability:
    - CONNECTION SIGHT (see encoding levels overlaid on reality)
    - Can create new meaning categories
    - Design god contracts (if approved by validators)

  lifestyle:
    - Creative class but surveilled
    - Valued but controlled
    - "Golden cage" existence

  dilemma:
    - Use power for system or against it?
    - Seeing connections = seeing suffering clearly
    - Many go mad from overdose of meaning
```

**MEANING WORKERS (10%)**

Create and maintain meanings. Writers, artists, coders, teachers.

```yaml
meaning_workers:
  job:
    - Generate content that increases system encoding
    - Maintain existing meanings (update, contextualize)
    - Flag "dangerous" meanings for review

  payment:
    - Encoding tokens (not money, but converts)
    - Health insurance NFT (if metrics met)
    - Housing allocation based on productivity

  surveillance:
    - Every creation logged
    - Plagiarism = instant demotion
    - "Original thought" score tracked
```

**CONSUMERS (40%)**

Use meanings but don't create. Most people.

```yaml
consumers:
  existence:
    - Consume meaning-products (entertainment, education, spirituality)
    - Vote on meanings (but votes worth little)
    - Work non-meaning jobs (physical, service, care)

  limitation:
    - Can't create new meanings (no permission)
    - Must use approved vocabulary
    - Creative expression requires license

  aspiration:
    - "Meaning worker exam" every 5 years
    - Some pass, most don't
    - System needs consumers more than creators
```

**GHOSTS (49%)**

Below threshold. System sees them but doesn't value them.

```yaml
ghosts:
  condition:
    - Encoding under 100 (born into it or fell)
    - Votes mathematically meaningless
    - Services deprioritized

  survival:
    - Black market meanings (unverified, risky)
    - Ghost communities (solidarity networks)
    - Some embrace invisibility (freedom in irrelevance?)

  danger:
    - One bad transaction = drop to BURNED status
    - No safety net
    - Children inherit ghost status (inter-generational trap)
```

**THE BURNED (uncounted)**

Encoding = 0. Wallet revoked. Legally non-existent.

```yaml
burned:
  how_to_become:
    - Massive debt default
    - "Meaning crimes" (creating illegal meanings)
    - Validator decision (no appeal)
    - System glitch (rare but happens)

  existence:
    - Can't transact (no wallet)
    - Can't prove identity
    - Can't access healthcare, housing, food legally

  resistance:
    - Some burned become "unchain" activists
    - Offline communities in dead zones
    - Rumored "fork" project to create alternative chain
```
