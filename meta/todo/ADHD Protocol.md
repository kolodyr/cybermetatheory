# 0. Мета протоколу

- Зняти відчуття «мене розриває» і замінити його на відчуття «мене тримає рейка».
- Дати мінімальний набір ритуалів, які працюють навіть коли Франко без сил.
- Прив’язати ADHD-флоу до структури cybermetatheory, а не до хаотичних нотаток.

---

# 1. Базові правила

1. Один день — один фокус.
   - День не може одночасно бути «про все».
   - На кожен день обирається одна домінантна зона:
     - [WORK] — заробіток, клієнти, конкретні дедлайни.
     - [ARCHIVE] — архів, впорядкування, meta_hata, запис сенсів.
     - [LEARN] — навчання, читання, курси, документація.
     - [WORLD] — організаційні справи, документи, медицина, бюрократія.

2. Два активні проєкти одночасно — максимум.
   - Один «хлібний» (гроші/навички для грошей).
   - Один «свій світ» (cybermetatheory / архів / мета-конструкти).
   - Усе інше автоматично йде в «інкубатор» (список/папка, але не в голову).

3. Нічого не тримається в голові довше 10 секунд.
   - Якщо думка не лягає в дію за 10 секунд — вона фіксується в inbox:
     - `meta/inbox.md` у репо, або
     - «Inbox» сторінка в Notion.
   - Голова не місце для TODO, тільки для орієнтації в контексті.

# 2. Ранковий протокол (10–20 хв)

trigger:
  - перша сесія за компом, не обов’язково зранку
goal:
  - не «зробити все», а обрати рейки на сьогодні

шаги:

1. Відкрити одну сторінку:
   - у файловій системі: `meta/today.md`, або
   - в Notion: сторінка «Сьогодні / Today».

2. Записати три речі:
   - MUST: одна річ, яка точно має статися сьогодні (навіть якщо малесенька).
   - JOY: одна річ, яка дає енергію / кайф (можна з cybermetatheory).
   - MIN: мінімум для тіла/нервової системи (сон, їжа, прогулянка, душ).

3. Обрати зону дня:
   - Написати вгорі: `zone: [WORK]` або `[ARCHIVE]` або `[LEARN]` або `[WORLD]`.
   - Це не забороняє інше, але дає пріоритет, коли мозок «зависає».

4. Обрати до 3 задач на день:
   - Кожна задача формулюється як маленький крок, який можна реально завершити.
   - Формат: `- [ ] [tag] короткий опис (де / що треба відкрити)`.
   - Приклад:
     - `- [ ] [WORK] Upwork: відповісти X, оновити профіль`
     - `- [ ] [ARCHIVE] Розкласти 10 нотаток в meta_hata`
     - `- [ ] [LEARN] Перечитати 1 статтю про git-flow`

# 3. Протокол для «мене розриває»

trigger:
  - багато бажань/страхів/ідей одночасно
  - неможливо обрати, за що взятися

file:
  - `meta/brain_dump_<YYYY-MM-DD>.md`

шаги (15–20 хв):

1. Brain dump (5–10 хв).
   - Відкрити файл `meta/brain_dump_<дата>.md`.
   - Виписати всі «хочу», «треба», «було би круто», «страшно забути».
   - Не структурувати, не фільтрувати, писати будь-яким стилем.

2. Маркування (5–10 хв).
   - Пройтися по списку і проставити теги:
     - `S` — survival: критичне (здоров’я, документи, дедлайни, житло).
     - `M` — money: дає гроші або збільшує шанс заробітку.
     - `E` — energy: дає енергію / кайф / сенс.
   - Одна задача може мати кілька тегів (наприклад `S+M`).

3. Вибір рейок:
   - Обрати:
     - 1 задачу з `S` або `S+M` → це перша конкретна дія.
     - 1 маленьку задачу з `E` → як нагорода після першої.
   - Перенести їх у `meta/today.md` в секцію задач сьогодні.

4. Архівація:
   - Усе, що ліниво/боляче бачити — можна перенести в окремий блок:
     - `## Інкубатор` в тому ж файлі.
   - Важливе — виноситься в окремі файли / сторінки, але тільки після виконання мінімуму з `S/M`.

# 4. Мікро-цикли роботи (ADHD-friendly)

format:
  - цикл «фокус» + «перерва»
  - можна 25/5 або 40/10, вибирає Франко під стан

правила:

1. Перед стартом:
   - Вибрати одну задачу з `meta/today.md`.
   - Прописати її вгорі файлу, з яким працюєш:
     - `focus: <короткий опис>`.

2. Під час циклу:
   - Заборонено міняти задачу, але можна міняти спосіб.
     - Напр.: якщо важко писати текст — можна зробити структуру, заголовки, коментарі.
   - Якщо прилітає нова ідея:
     - Записати одним рядком в `meta/inbox.md`.
     - Повернутися до поточної задачі.

3. Коли час вийшов:
   - Поставити собі мітку:
     - `done: X хвилин` і що саме зроблено.
   - Зробити коротку перерву (встати, відійти від екрану, подихати).

4. Після 3 циклів:
   - Довша перерва (20–30 хв), без скролу соцмереж, по можливості без екранів.

# 5. Вечірній протокол (5–10 хв)

trigger:
  - перед сном / перед вимиканням компа

file:
  - `meta/today.md`

шаги:

1. Закрити день:
   - Позначити зроблене:
     - `[x]` для завершених задач.
     - `[-]` для частково зроблених, з коротким коментарем.
   - Те, що не зроблено:
     - або перенести на завтра свідомо,
     - або чесно викреслити, якщо воно не варте енергії.

2. Записати короткий лог:
   - 1–3 рядки про те, що дало енергію.
   - 1–3 рядки про те, що забрало енергію.
   - Приклад:
     - `+ живий чат з X`
     - `- завис у браузері замість короткої перерви`

3. Підготовка завтра:
   - Якщо є сили — чернеткою набросати MUST/JOY/MIN на завтра.
   - Якщо немає сил — хоча б написати: `завтра фокус: [WORK]/[ARCHIVE]/[LEARN]/[WORLD]`.

# 6. Інтеграція з black_swan_protocol

- `adhd_protocol.md` відповідає на питання «як рухатися, коли Франко живий, але розфокусований».
- `black_swan_protocol.md` відповідає на питання «що робити, коли Франко лежить і у нього мінімум сил / криза / чорний лебідь».
- У кожному файлі можна тримати посилання один на одного:
  - в `black_swan_protocol.md`:
    - `see also: meta/adhd_protocol.md (режим, коли сил вже трохи більше)`
  - в `adhd_protocol.md`:
    - `fallback: meta/black_swan_protocol.md (коли нічого не тягне, окрім виживання)`

---

# 7. Право переписувати

- Цей протокол не догма, а живий артефакт.
- Раз на тиждень можна відкривати цей файл і:
  - видаляти те, що не працює,
  - додавати свої приклади і мікро-ритуали,
  - відмічати «ядерні» речі, які реально допомагають.
- Головне правило: протокол завжди має бути достатньо простим, щоб спрацювати навіть у поганий день.
