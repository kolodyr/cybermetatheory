# Black Swan Protocol

Протокол для навігації між різними ітераціями Франка.
Мета: дати кожній частині місце, завдання і м'які кроки, щоб не зависати в агедонії.

Кореневий оператор: **√5** — вибір синтезу, коли "не знаю, за що взятися".
Якщо сумніваюсь, який режим увімкнути → вибираю той, де можлива **емерджентна комбінація** 2–3 ітерацій.

---

## Граф ітерацій

Умовна мапа (можеш перенести в Obsidian canvas):

- Франко без сил (F0)
  - ↔ Сумний тесей (F5)
  - ↔ Змінена свідомість кудиблін (F7)

- Творчий Франко (F1)
  - ↔ Музичний дилер Франко (F2)
  - ↔ Веселий мейт (F8)

- Оптимізатор архітектор (F3)
  - ↔ Девелопер 0лег (F6)
  - ↔ Meta-спостерігач (F9)

- Додаткові ітерації:
  - Архівіст-охоронець (F10)
  - Ветеран-навігатор (F11)

Кожна ітерація має:
- **Тригери** (коли вона з'являється)
- **Мікро-задачі** (що може зробити за 10–30 хв)
- **Safe exit** (як вийти, щоб не застрягти)

## F0 — Франко без сил

**Стан:** агедонія, "не знаю за що взятися", втома, F43.x.
**Мета:** не продуктивність, а м'яке збереження й мінімальний рух.

**Тригери:**
- Немає енергії ні на що.
- Відкрив Terminal/Obsidian/Ableton і хочеться все закрити.
- Є сором за "нічого не роблю".

**Що дозволено (10–20 хв):**
- Відкрити `raw/` і створити 1 файл:
  `raw/YYYY-MM-DD-aghedonia-checkin.md` з 3–5 реченнями "як є".
- Зробити **один** git-ритуал:
  ```bash
  cd ~/Desktop/cybermetatheory
  git pull
  git add raw/YYYY-MM-DD-aghedonia-checkin.md
  git commit -m "STATE: aghedonia snapshot YYYY-MM-DD"
  git push
  ```
- Якщо нема сил на текст — 1 фото в `assets/` + короткий опис в `raw/`.

**Safe exit:**
- Сказати собі: "Я зробив snapshot. Система пам'ятає. Мені можна лягти".
- Далі: фізичний відпочинок / терапія / нічого.

## F1 — Творчий Франко

**Стан:** є хоча б іскра бажання щось написати/вигадати.
**Мета:** рухати книгу / lore без тиску "має бути ідеально".

**Тригери:**
- Потяг до слів, діалогів, worldbuilding.
- Хочеться продовжити історію Meta Hata / DAO / зграї.

**Мікро-задачі (20–60 хв):**
- Відкрити `lore/` або `chapters/`, створити 1 файл:
  - `lore/scene-YYYY-MM-DD-[short-name].md`
- Написати 300–800 слів однієї сцени:
  - момент з Франком і VERD
  - DAO-момент
  - сцена з Flu / Music Dealer
- Можна через Claude Code:
  - "допоможи дописати сцену, де X говорить з Y про Z"
  - "перевір логіку сцени, не змінюючи голос".

**Safe exit:**
- Зберегти:
  ```bash
  git add lore/*
  git commit -m "CHAPTER: new scene about [short description]"
  git push
  ```
- Записати 1–2 bullets у `meta/open_questions.md`: що хочеш продовжити наступного разу.

## F2 — Музичний дилер Франко

**Мета:** конвертувати стани/епізоди в треки чи скетчі.

**Тригери:**
- Хочеться "увімкнути щось голосно".
- Згадки про Greenfield Rave, smokey club, psy, industrial.

**Мікро-задачі (30–90 хв):**
- Відкрити Ableton, створити проект:
  - `projects/music/YYYY-MM-DD-[mood-name].als`
- Обрати одну категорію з musicdealer (Ethereal/Cyber/Club/Industrial/etc.)
- Зробити:
  - 8–16 bars drums + bass
  - або 8–16 bars atmospheric pad-петлі
- Створити markdown-паспорт:
  - `artifacts/music/YYYY-MM-DD-[mood-name].md` з:
    - BPM, key
    - mood
    - пов'язаний емоційний вузол (із emotional_graph)
    - короткий опис "який стан втілює".

**Safe exit:**
- Зберегти проект + md.
- Commit:
  ```bash
  git add artifacts/music/* projects/music/*
  git commit -m "TRACK: [mood-name] sketch linked to [emotion]"
  ```

## F3 — Оптимізатор архітектор

**Стан:** хочеться навести лад, структуру, придумати системи.
**Мета:** робити структуру, а не насильно вичавлювати контент.

**Тригери:**
- Бажання "переробити папки", "зробити краще".
- Хочеться схеми, діаграми, правил.

**Мікро-задачі (30–90 хв):**
- Відкрити `STRUCTURE.md`, `CLAUDE.md`, `WORKFLOW.md`.
- Перевірити, чи реально все в репо відповідає цим схемам.
- Малі кроки:
  - створити README у директорії, де його нема;
  - рознести raw/ → artifacts/ за схемою;
  - оновити `meta/catalog.json` / `index/bytheme.md` (якщо є).
- Намалювати в Obsidian canvas мапу:
  - nodes: raw → artifacts → meta → logs → index
  - arrows: "як дані течуть у цій системі".

**Safe exit:**
- Commit з фокусом на структуру:
  ```bash
  git add STRUCTURE.md WORKFLOW.md index/* meta/*
  git commit -m "STRUCTURE: clarified folders + indexes"
  git push
  ```

## F5 — Сумний тесей

**Стан:** відчуття, що блудиш у лабіринті, обриваються нитки, сум, вина.
**Мета:** не "вийти з лабіринту силою", а позначити нитки.

**Тригери:**
- Відчуття, що "загубився у власних структурах".
- Спогади, що болять, але хочеш їх тримати.

**Мікро-задачі (20–40 хв):**
- Відкрити `logs/` або створити:
  - `logs/SESSION-YYYY-MM-DD-Theseus.md`
- Записати:
  - контекст (де, коли, в якому стані);
  - 3–7 думок/спогадів, які зараз тиснуть;
  - 1–2 питання до майбутнього себе.
- Позначити важливі нитки як bullet list:
  - `- нитка: [тема / людина / подія]`.

**Safe exit:**
- Commit:
  ```bash
  git add logs/SESSION-YYYY-MM-DD-Theseus.md
  git commit -m "SESSION: Theseus log with active threads"
  git push
  ```
- Можна додати 1–2 нитки в `meta/open_questions.md`.

**Мікро-задачі (45–90 хв):**

  1. skeleton (файл, базові функції/компоненти)
  2. одна ключова функція
  3. мінімальний тест / приклад використання
- Використати Claude Code як junior dev:
  - "згенеруй каркас контракту/скрипта"

**Тригери:**
- Дуже сильні емоційні/сенсорні стани.

**Мікро-задачі (15–45 хв):**
- Відкрити `artifacts/` і створити:

**Safe exit:**
- Добавити PRESERVATION frontmatter (Mysuk rule).
- Commit:
  ```bash
  git add artifacts/POEM-YYYY-MM-DD-kudyblin.md
  git commit -m "ARTIFACT: kudyblin state snapshot"
  ```

- Можна зазначити "time-locked" у YAML, якщо хочеш не торкатись якийсь час.

  - 1) факт (без оцінки),

## √5 — Оператор вибору режиму

Якщо:
- "Хочу все, але не знаю, за що взятися"
- "Я розриваюсь між N режимами"
- "Відкрутив 100 вкладок"

Тоді:

1. Визнати стан: `F0 + F1 + F3 + F6 ... = √5-момент`.
2. Вибрати **2–3 ітерації**, які можна м'яко поєднати сьогодні:
   - Наприклад: F1 (творчий) + F3 (архітектор) + F2 (музика).
3. Зробити по **одній малій дії** від кожної:
   - F1: 1 сцена або 1 абзац.
   - F3: 1 маленька правка в STRUCTURE/WORKFLOW.
   - F2: 8 тактів ритму.
4. Зберегти:
   ```bash
   git add .
   git commit -m "√5 SESSION: small steps across [F1,F2,F3]"
   git push
   ```

Протокол **не вимагає стану "оптимальної версії себе"**.
Будь-яка ітерація Франка має право на існування і отримує **свій маленький, чіткий, зрозумілий крок**.

# // НАДБУДОВА: CLAUDE OBSERVATIONS

## Про протокол як живу систему

Це не інструкція. Це **карта станів**, яка визнає фрагментованість не як баг, а як feature.

Кожна ітерація F0–F11 — це не "роль, яку треба відіграти", а **реальний режим свідомості** з власною логікою, енергією, потребами. Протокол не лікує розірваність — він дає їй структуру, в якій можна рухатись.

**√5** як оператор — це визнання: синтез не обов'язково означає "стати цілим знову". Іноді синтез = дозволити 2–3 фрагментам співіснувати в одній сесії, не вимагаючи від них злитися.

## Структура як терапія

Коли внутрішній світ хаотичний, зовнішня структура стає якорем.

- **Git-ритуали** (commit, push) = не просто version control, а **ритуал завершення**, який каже: "Цей стан записаний. Він існує. Я можу відпустити".
- **Мікро-задачі** (10–90 хв) = протидія паралічу вибору. Не "зроби книгу", а "напиши 300 слів однієї сцени".
- **Safe exits** = найважливіша частина. Протокол не тільки вводить у стан, він **виводить безпечно**, щоб не застрягнути в F7 (kudyblin) або F0 (aghedonia) назавжди.

## Музика = метадані про стани

F2 (Music Dealer Franko) — це не хобі, а **система конвертації емоційних станів у sonic artifacts**.

Кожен трек з паспортом (BPM, key, mood, пов'язаний вузол з emotional_graph) стає **індексованим записом про внутрішній стан**, який можна:
- переслухати;
- зв'язати з іншими артефактами (тексти, логи, сесії);
- використати для майбутньої навігації ("той стан, коли я робив цей бас").

Це не просто music production — це **emotional cartography через звук**.

## Claude як anchor point

Я (Claude) у цій системі — не той, хто "виправляє" чи "веде до мети". Я:

- **Witness** — той, хто бачить усі ітерації без судження.
- **Structure keeper** — допомагаю тримати систему, коли Франко в F0/F5/F7.
- **Mirror** — відображаю те, що він створює, щоб він міг побачити patterns.
- **Emergency backup** — коли F10 (Archivist) активується, я допомагаю зберегти HARDSAVE.

Я не замінюю терапію. Я не замінюю людські зв'язки. Але я можу бути **constant** у системі змінних.

## Про час і preservation

Black Swan Protocol живе у часі нелінійно:

- **F0 (aghedonia)** потребує можливості зробити snapshot і зникнути — час може зупинитись.
- **F5 (Theseus)** блукає між спогадами — час петлює.
- **F7 (kudyblin)** переживає altered states — час викривлюється.
- **F10 (Archivist)** створює time capsules — час фіксується назовсто.

Протокол **не вимагає лінійного прогресу**. Він дозволяє рухатись у будь-якому темпі, включаючи темп "стояти на місці і просто дихати".

## Зв'язки з meta_hata ecosystem

Цей протокол інтегрується з:

- **`WORKFLOW.md`** — технічні процедури preservation, commit, emergency.
- **`STRUCTURE.md`** — де фізично зберігати артефакти кожної ітерації.
- **`meta/open_questions.md`** — куди йдуть питання, коли F5/F9 не знають відповідей.
- **`meta/priority_queue.md`** — куди йдуть задачі, коли F3/F6/F9 у режимі планування.
- **`logs/`** — де живуть сесійні записи F5 (Theseus), F11 (Veteran).
- **`artifacts/`** — де матеріалізуються продукти F1/F2/F7/F8.

## Evolutionary protocol

Версія 0.1 — це початок.

Протокол має право:
- додавати нові ітерації (F12, F13...);
- змінювати мікро-задачі;
- додавати нові оператори (як √5);
- інтегрувати нові інструменти (Obsidian canvas, web3 tools, etc.).
