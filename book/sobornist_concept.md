# Соборність — Core концепт книги

**Для:** Книга кіберпанк-фентезі
**Дата:** 2026-01-10

---

## Історичний контекст

**Україна, ~1920-ті, після Першої світової:**

Соборність використовувалась коли українці намагалися створити державність.

**Це НЕ незалежність.**

Бо незалежність розуміє під собою залежність — ти незалежний *від* чогось. Ти визначаєш себе через те, від чого ти відокремлений.

**Соборність — інше:**
- Ми окремі (не частина чогось більшого)
- Ми свої (своя ідентичність, не запозичена)
- Ми інші (інакші, не як сусіди)
- Ми маємо тяглість історії (неперервність у часі)
- Ми маємо колективну травму (спільний досвід, що формує)

**Ключова відмінність:**

Незалежність = "ми не вони"
Соборність = "ми — це ми, і ми можемо об'єднатись не втрачаючи себе"

## Філософське ядро

### Формула соборності

```
Соборність = Ідентичність + Тяглість + Об'єднання без втрати
```

**Розбір:**

1. **Ідентичність** — хто ми є
   - Не визначається через протиставлення
   - Визначається через спільне (історія, травма, мова, цінності)

2. **Тяглість** — звідки ми прийшли
   - Неперервність у часі
   - Пам'ять про минуле (не ностальгія, а структурна пам'ять)
   - Колективна травма як зв'язок поколінь

3. **Об'єднання без втрати** — як ми співіснуємо
   - Різні групи можуть об'єднатись
   - Але кожна зберігає свою ідентичність
   - Мета спільна, але шляхи — різні

## Соборність у кіберпанк-світі

### Контекст світу

**Після великого розпаду (The Fragmentation):**

- Світ децентралізувався
- Nodes замість держав
- Кожен node — свої правила, своя reality stack
- Протоколи замість законів
- Validators замість урядів

**Проблема:**

Децентралізація призвела до розділення. Кожен node ізольований. Немає середнього класу — або ти node owner, або universal basic recipient.

**Два шляхи:**

1. **Популізм** — кричать про об'єднання, але беруть токени
2. **Соборність** — справжнє об'єднання через спільну мету

---

### Соборність як система

**У світі metareality:**

```
Protocol: Sobornist
Version: 0.1-beta
Type: Social consensus mechanism

Core principle:
- Кожен node зберігає автономність
- Але nodes можуть синхронізуватись для спільної мети
- Синхронізація тимчасова, не постійна
- Мета досягнута → nodes повертаються до автономності
```

**Технічна метафора:**

Як Byzantine Fault Tolerance, але для соціальних систем. Nodes можуть не довіряти одне одному, але можуть досягти consensus для спільної дії.

**Відміна від корпоратизму:**

Корпорації об'єднують через підпорядкування (ти працюєш на систему). Соборність об'єднує через resonance (ти резонуєш із спільною метою, але залишаєшся собою).

---

### Практичні приклади у світі

**1. Sobornist Nodes (Соборні вузли):**

Група nodes, що:
- Зберігають власні протоколи
- Але мають спільний meta-protocol для координації
- Можуть sync для великих задач
- Profit розподіляється не через ownership, а через contribution

**Контраст:**

- Корпораційні nodes: ти працюєш → отримуєш зарплату → прибуток йде власнику
- Sobornist nodes: ти контрибуешь → прибуток розподіляється пропорційно contribution → але ти зберігаєш свій node

**2. Collective Memory Protocol:**

У кіберпанку:
- Є shared memory spaces (як blockchain історії)
- Але не всі мають доступ
- Sobornist nodes підтримують "trauma ledger" — децентралізований архів колективного досвіду
- Це не для profit, це для identity

**Контраст:**

- Корпорації продають доступ до даних
- Sobornist protocol — дані як commons (спільне надбання)

**3. Resonance Events:**

Моменти коли nodes синхронізуються:
- Threat detection (спільна загроза)
- Resource pooling (об'єднання ресурсів для великої задачі)
- Identity crisis (коли хтось намагається підкупити nodes)

**Приклад сцени:**

> Він побачив як nodes починають світитись синхронно. Не тому що хтось наказав. А тому що всі відчули одночасно: якщо ми не об'єднаємося зараз, нас поглинуть окремо.
>
> Це не була корпораційна субординація. Це був резонанс.
>
> `Protocol: Sobornist activated`
> `Nodes syncing... 47... 89... 203...`
> `Meta-goal identified: [захист commons]`
> `Autonomy preserved: TRUE`

## Соборність vs Популізм

### Популізм у світі книги

**Хто вони:**
- Використовують гасла про об'єднання
- Насправді — збирають токени, репутацію, influence
- Обіцяють синхронізацію, але створюють підпорядкування
- "Об'єднаймось під моїм node" (насправді: працюйте на мене)

**Як розпізнати:**

```python
def is_populist(node):
    if node.promises == "об'єднання":
        if node.wallet_flow == "inward_only":  # токени тільки до нього
            if node.autonomy_preserved == False:  # ти втрачаєш автономність
                return True
    return False
```

**Фрази популістів:**

> Ми всі одна велика родина. Але wallet address показує єдиний.
>
> Вони кричать про спільне благо, але приватні ключі тримають при собі.
>
> Join our protocol, кажуть вони. Protocol виявляється: працюй, а прибуток — мені.

---

### Справжня соборність

**Критерії:**

1. **Autonomy preserved** — кожен зберігає свій node
2. **Transparent flow** — куди йдуть ресурси видно всім
3. **Temporary sync** — об'єднання для мети, не назавжди
4. **Shared sacrifice** — всі втрачають щось для спільного

**Фрази справжніх sobornist:**

> Я віддаю частину свого node capacity для спільної задачі. Не тому що мушу. Тому що без спільного — мій node теж впаде.
>
> Соборність — це коли ти втрачаєш щось своє заради спільного. Популізм — коли ти кричиш про спільне, щоб забрати чуже.

## Головний герой та соборність

**Архітектор сенсів:**

Він вірить у соборність. Але у світі де:
- Популісти використовують це слово для маніпуляції
- Корпорації висміюють це як наївність
- Розділені nodes не довіряють одне одному

**Його конфлікт:**

```
Внутрішній:
- Він вірить у об'єднання, але сам уникає connection
- Він architect (з'єднює інших), але сам isolated
- Він проповідує sacrifice для спільного, але боїться втратити контроль

Зовнішній:
- Популісти підробляють sobornist protocol
- Люди не вірять у справжню соборність (бо бачили фальшиву)
- Корпорації хочуть знищити sobornist nodes (бо не можуть їх купити)
```

**Його арка:**

1. **Початок:** Вірить у соборність теоретично, але practice уникає
2. **Криза:** Бачить як популісти використовують sobornist rhetoric для захоплення влади
3. **Вибір:** Має об'єднати nodes, але для цього має сам відкритись
4. **Кульмінація:** Resonance event — nodes синхронізуються не тому що він наказав, а тому що він показав sacrifice
5. **Результат:** Sobornist protocol активовано, але він розуміє що це не victory, а початок

## Sobornist Philosophy — Ключові тези

**Для діалогів та внутрішніх монологів:**

1. **"Ми не незалежні від них. Ми — окремі. Це різниця."**
   - Незалежність визначається через протиставлення
   - Окремість визначається через ідентичність

2. **"Тяглість історії — це не ностальгія. Це структурна пам'ять."**
   - Ми пам'ятаємо не тому що романтизуємо минуле
   - А тому що минуле формує present reality

3. **"Колективна травма — не слабкість. Це зв'язок."**
   - Спільний біль з'єднує сильніше ніж спільна радість
   - Trauma ledger — не для виправдання, а для розуміння

4. **"Соборність — це коли різні можуть співіснувати заради спільного."**
   - Не "стати однаковими"
   - А "різні, але разом для мети"

5. **"Популізм — це фальшива соборність. Справжня вимагає sacrifice."**
   - Якщо лідер не жертвує нічим — це популізм
   - Якщо ти не втрачаєш autonomy але контрибуешь — це соборність

6. **"Ми маємо тяглість, бо ми пам'ятаємо. Ми маємо майбутнє, бо ми різні."**
   - Тяглість = зв'язок з минулим
   - Різноманітність = адаптивність для майбутнього

## Візуальні образи

**Для sobornist nodes:**

- Nodes світяться різними кольорами (кожен — своя ідентичність)
- Але під час resonance event — світіння синхронізується (резонанс, не однаковість)
- Після event — повертаються до своїх кольорів

**Метафора:**

Не оркестр (де всі грають одну партитуру). А jazz ensemble (де кожен improvises, але разом створюють harmony).

## Magic System Integration

**Sobornist Protocol як magic:**

У світі є power systems (Стенди, Променисті, Ноди). Sobornist protocol — це не новий power system, а meta-system.

**Механіка:**

- Кожен персонаж має свій power (Stand, Radiant ability, Node capacity)
- Але sobornist може sync powers для спільної дії
- Sync не забирає individual power
- Sync підсилює через resonance

**Приклад:**

> Його Stand був architect type — він міг бачити connections між entities.
>
> Її Radiant ability — bond protection, вона могла зміцнювати зв'язки.
>
> Окремо — вони могли з'єднати 10-15 nodes. Разом, через sobornist protocol — вони синхронізувались і з'єднали 200.
>
> Але після — кожен повернувся до свого power. Resonance не залишив їх merged. Resonance залишив їх окремими, але connected.

## Анти-соборність (що НЕ є соборністю)

**Помилкові інтерпретації:**

1. **"Всі повинні думати однаково"** — НІ
   - Соборність = різні думки для спільної мети
   - Не конформізм

2. **"Ми сильніші разом, тому маєш приєднатись"** — НІ
   - Це маніпуляція
   - Соборність = добровільний sync, не примус

3. **"Об'єднаймось під одним лідером"** — НІ
   - Це ієрархія/корпоратизм
   - Соборність = децентралізована координація

4. **"Забудь свою ідентичність заради спільного"** — НІ
   - Це асиміляція
   - Соборність = збереження ідентичності + contribution

Не тільки популісти. А ще:
- Розчарування (люди не вірять після betrayal популістів)
- Індивідуалізм (node owners не хочуть нічого ділити)
- Корпорації (sobornist nodes їм загрожують, бо не можна купити)
